# OkiMail

source: bot-spec
updated: 2026-04-26
category: bot

## 概要

OkiMail は、発行したメールアドレスに届いたメールを Discord の DM に転送する Bot です。

メール本文を抽出して Discord で表示可能な形式に変換し、利用者の DM へ送信します。

本文だけでなく、送信されたメールファイル（`.eml`）も添付されます。

画像などの添付ファイルも転送できます。

ただし、ファイル容量が大きい場合は、メールファイルや添付の一部が省略される場合があります。

## 主な機能

- OkiMail アドレス宛てメールの自動受信
- メール本文の抽出とデコード
- Discord DM への自動転送
- `.eml` メールファイルの添付
- 画像や ZIP、ソースコードなどの添付ファイル転送
- 送信元メールアドレス単位のブロック

## ブロック機能

OkiMail には、送信元アドレス単位のブロック機能があります。

ブロックした送信元からのメールは、DM に届かず通知もされません。

ブロック機能はユーザー ID 単位で管理されます。

そのため、複数の OkiMail アドレスを所持している場合でも、同一ユーザーとして一括でブロックが適用されます。

## 重要な位置づけ

OkiMail は、遊び用途や検証用途を想定したメール転送 Bot です。

セキュリティ上の欠陥が存在する可能性があるため、メインのメールアドレスとしての利用は推奨されません。

## 利用手順

1. 発行依頼チャンネルで OkiMail アドレスの発行を依頼する
2. OkiMail アドレスを受け取る
3. 外部サービスや検証用途で利用する
4. OkiMail がメールを受信する
5. OkiMail が本文を抽出・デコードする
6. Discord の DM に、本文・メールファイル・添付が転送される

受信は完全自動です。

利用者側で追加設定は不要です。

## 登録完了通知が届かない場合

DM や年齢設定などの影響で届かない場合があります。

その場合は、OkiMail Bot の「アプリを追加」を押したうえで、発行依頼チャンネルにて再発行を申し出てください。

OkiMail Bot のユーザー ID は `1446422325594554482` です。

## 想定用途

次のような用途を想定しています。

- 外部サービスの検証用
- X などの SNS アカウント作成
- wplace、wiki、サーバー運営に関連する連絡用
- 比較的リスクのあるサイトへの登録
- 各種サービスのログイン通知や障害通知の受信用

これらはあくまで例です。

犯罪などに使用しないでください。

重要な個人情報を扱う用途や、メインのメールアドレスとしての利用は推奨しません。

## コマンド

OkiMail は、送信元アドレス単位のブロック機能を備えています。

ブロックはユーザー ID 単位で管理されるため、複数の OkiMail アドレスを持っていても、同一ユーザーとして一括で適用されます。

ブロックした送信元からのメールは、DM に届かず通知もされません。

### /block

`/block` は、指定した送信元メールアドレスからの受信を拒否するコマンドです。

```text
/block <メールアドレス>
```

### /unblock

`/unblock` は、指定した送信元メールアドレスのブロックを解除するコマンドです。

```text
/unblock <メールアドレス>
```

### /id

`/id` は、自分の OkiMail アドレス一覧を確認するコマンドです。

```text
/id <ユーザーID>
```

このコマンドは、本人または管理チャンネルのみ有効です。

ユーザー ID を確認するには、Discord のユーザー設定から詳細設定を開き、開発者モードを ON にしてください。

## 利用上の注意

OkiMail は、遊び用途や検証用途を想定したサービスです。

ドメインを更新しない場合があるため、1年後の継続利用は確約されません。

定期的な確認は行われていますが、セキュリティ上の欠陥が存在する可能性があります。

メインのメールアドレスとしては使わず、あくまで遊び用途で利用してください。

使用用途によっては、利用停止措置が取られる場合があります。

犯罪幇助などにつながる用途では利用しないでください。

## 文字化け

受信したメールは本文を抽出して送信します。

ただし、メールの形式やエンコード方式によっては、一部の特殊なメールで文字化けが発生する場合があります。

## 添付ファイルの制限

画像を含む添付ファイルは、本文への埋め込み・通常添付を問わず転送可能です。

ただし、Discord 側の仕様により、メールファイルを含めた添付は 10MiB までです。

また、Cloudflare の制約により、処理サーバーでの受信時・送信時の双方で合計 25MiB のサイズ制限があります。

これらの制限を超過する場合、メールファイルや添付の一部が省略される場合があります。

## 容量超過が起きやすい場面

受信時は、元のメールファイルに加えて添付を抽出して送信します。

そのため、合計容量を超えやすい場合があります。

添付が省略される場合は、送信を複数回に分けてもらうなどしてください。

容量制限は、画像、ZIP、コードなどのファイル種類には依存しません。

## サーバー脱退時の利用

OkiMail Bot を導入しているサーバーに加入していない場合、Bot の権限が届きません。

そのため、サーバーを抜けるとメールの受信ができなくなります。

### OkiMail から返信や送信はできますか？

できません。

OkiMail は、受信したメールを Discord に転送する機能のみ対応しています。

### 画像以外のファイルも届きますか？

はい。

画像は、本文埋め込み・通常添付のどちらも転送可能です。

ソースコードや ZIP などのファイルも受信可能です。

ただし、中身には十分注意し、信頼できる送信者のもののみ開くようにしてください。

サイズ制限により、省略される場合もあります。

### 添付が省略されるのはどんな場合ですか？

Cloudflare の合計 25MiB 制限、または Discord の単体 10MiB 制限を超える場合などです。

送信時・受信時ともに、Cloudflare の合計 25MiB 制限が適用されます。

ファイル単体が 10MiB を超える場合、一部またはすべてのファイルが省略されることがあります。

### 一部のメールが文字化けするのはなぜですか？

メールの形式やエンコード方式が特殊な場合、デコード処理の都合で文字化けが起きることがあります。
